# 修复记录

## 修复 #001 | 2026-04-18

### 问题
**API-账户余额 获取失败**
- **原因:** `ccxt.binance` 使用 `defaultPositionSide: 'LONG'` 参数初始化后，`fetch_balance()` 返回的 USDT 字段为空 `{}`
- **现象:** 健康检查时余额显示为 0，但实际账户有余额

### 根因
`defaultPositionSide: 'LONG'` 是币安U本位永续合约专用参数，会改变 API 请求的行为模式。当此参数存在时，`fetch_balance()` 的返回结构中 USDT 键为空字典。

### 修复方案
创建独立的 binance 客户端实例用于查询余额，不携带 `defaultPositionSide` 参数：

```python
# 修复前（有问题）
binance = ccxt.binance({
    'apiKey': API_KEY,
    'secret': SECRET,
    'options': {'defaultType': 'swap', 'defaultPositionSide': 'LONG', 'marginMode': 'cross'}
})
r = binance.fetch_balance()  # USDT 为空 {}

# 修复后（正常）
bal_binance = ccxt.binance({'apiKey': API_KEY, 'secret': SECRET, 'options': {'defaultType': 'swap'}})
r = bal_binance.fetch_balance()  # USDT: {'free': 110.13, 'total': 110.13}
```
